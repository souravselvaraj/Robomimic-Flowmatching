# robomimic-cfm

**Conditional Flow Matching (CFM) policy for [robomimic](https://github.com/ARISE-Initiative/robomimic)** —
a standalone package that adds a rectified-flow / OT-CFM policy as a drop-in
alternative to Diffusion Policy. It layers on top of robomimic rather than
forking it: importing the package registers a `"flow_matching"` algorithm with
robomimic's factories, after which it works exactly like a built-in algo.

It reuses robomimic's observation encoder and `ConditionalUnet1D` backbone, but
instead of learning to denoise it learns a **velocity field** `v(x_t, t, obs)`
along the straight-line path from Gaussian noise to action sequences:

```
x_t = (1 - (1 - σ_min)·t)·x₀ + t·actions      # linear interpolant, x₀ ~ N(0, I)
u_t = actions - (1 - σ_min)·x₀                # target velocity
loss = ‖ v_θ(x_t, t, obs) - u_t ‖²
```

At test time actions are generated by integrating the learned ODE with just
**10 Euler steps** (vs. 100 DDPM denoising steps), receding-horizon style exactly
like Diffusion Policy.

## Installation

```bash
git clone https://github.com/souravselvaraj/Robomimic-Flowmatching.git
cd Robomimic-Flowmatching
uv sync
```

`uv sync` reproduces the exact locked environment from `uv.lock`, including
robomimic itself — the package depends on the robomimic fork it was developed
against (pinned by commit), because the diffusion-policy backbone it builds on
predates the latest robomimic release on PyPI. No extra steps are needed.

<summary>Installing with pip instead of uv</summary>

```bash
pip install "robomimic @ git+https://github.com/souravselvaraj/robomimic@562c8e323485391a049b67be41990f527b0f07a2"
pip install -e .
```

Rollout-video rendering and the reproduction scripts pull a few extra libraries:
`uv sync --extra extras` (or `pip install -e ".[extras]"`).

## Quickstart

Importing the package is all it takes to register the algorithm:

```python
import robomimic_cfm                       # registers "flow_matching"
from robomimic.config import config_factory
config = config_factory(algo_name="flow_matching")
```

Train and evaluate on a robomimic task (downloads via robomimic's own tooling):

```bash
# 1. download a dataset (standard robomimic way)
python -m robomimic.scripts.download_datasets --tasks lift --dataset_types ph --hdf5_types low_dim

# 2. train + evaluate through the full robomimic pipeline
python scripts/train.py --config configs/benchmark/fm_lift_seed1.json

# 3. (optional) render a rollout video — headless, software OSMesa, no X/EGL
MUJOCO_GL=osmesa PYOPENGL_PLATFORM=osmesa \
    python scripts/render_video.py --ckpt <path-to-checkpoint>.pth --dataset <dataset>.hdf5
```

`scripts/train.py` imports `robomimic_cfm` (registering the algorithm) and then
hands off to robomimic's own trainer, so the entire robomimic training pipeline —
checkpointing, rollouts, resume — works unchanged. Adjust dataset/output paths in
the configs for your machine (see [configs/README.md](configs/README.md)).

Run the test suite (trains 5 variants end-to-end on CPU in a few minutes):

```bash
python tests/test_flow_matching.py
```

## Results (low_dim, proficient-human datasets)

**Success rate vs. Diffusion Policy**, matched budget (1000 epochs × 100
steps/epoch, batch 256), same encoder + `ConditionalUnet1D` backbone.
**Mean ± std over 3 seeds**, best-checkpoint on rollout success (n = 20 rollouts
per eval → 5% resolution). CFM uses 10 Euler steps; DP uses DDPM-100.

| Task | Flow Matching | Diffusion Policy | BC-RNN ref |
|---|---|---|---|
| Lift | **100.0 ± 0.0** | 100.0 ± 0.0 | ~100 |
| Can | **100.0 ± 0.0** | 100.0 ± 0.0 | ~100 |
| Square | **56.7 ± 8.5** | 45.0 ± 7.1 | ~84 |
| Tool Hang | 75.0 ± 7.1 | **80.0 ± 7.1** | ~67 |
| Transport | 81.7 ± 2.4 | **86.7 ± 8.5** | ~71 |

CFM is competitive with Diffusion Policy across the board — matched on the easy
tasks, clearly ahead on **Square** (the hardest precision task, +11.7 pts), and
slightly behind on Tool Hang / Transport.

**Inference speed** — per action-chunk latency on the identical 65M-param UNet
(RTX PRO 6000, 100 trials):

| Sampler | Latency (ms/chunk) | Speedup vs DDPM-100 |
|---|---|---|
| FM Euler, 1 step | **3.11 ± 0.02** | 94.4× |
| FM Euler, 5 steps | 13.44 ± 0.09 | 21.9× |
| FM Euler, 10 steps *(default)* | 26.68 ± 0.14 | **11.0×** |
| FM midpoint, 5 steps | 26.92 ± 0.14 | 10.9× |
| DP DDIM, 10 steps | 28.91 ± 0.10 | 10.2× |
| DP DDPM, 100 steps | 293.72 ± 0.61 | 1.0× |

CFM is **11× faster** than the standard DDPM-100 sampler at its default quality
setting. Against DDIM-10 (same number of network evaluations) it is ≈wall-clock
parity — its edge there is quality retained at few steps, not raw speed.

Reproduce any point with the matching config under [configs/](configs/) (all 30
runs of the sweep are included).

## Layout

```
robomimic_cfm/
  __init__.py            # import-time registration of the algo + config
  flow_matching.py       # FlowMatchingUNet (PolicyAlgo), registered as "flow_matching"
  config.py              # FlowMatchingConfig (BaseConfig, ALGO_NAME="flow_matching")
  exps/templates/flow_matching.json   # generated config template
scripts/                 # train.py (wrapper) + eval / render / inference-benchmark
tests/test_flow_matching.py           # 5 variants through train.py + reload + rollout
configs/                 # per-task + 3-seed benchmark configs (reproduction)
docs/CONDITIONAL_FLOW_MATCHING.md     # design notes
```

## How it works

robomimic's algorithm and config registries are plain, overwrite-safe
dictionaries. `robomimic_cfm` uses robomimic's own
`register_algo_factory_func` decorator and `ConfigMeta` metaclass, so simply
importing the package registers `"flow_matching"` on top of any robomimic
install — no changes to robomimic required. See
[docs/CONDITIONAL_FLOW_MATCHING.md](docs/CONDITIONAL_FLOW_MATCHING.md) for the
design and objective.

Note: actions must be normalized to `[-1, 1]` (standard robomimic datasets
already are). The cosine LR schedule uses `step_every_batch=True` — if you write
your own training loop, step the scheduler **per gradient step** or the LR stays
at 0 through warmup.
