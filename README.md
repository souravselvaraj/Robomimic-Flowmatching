# robomimic-cfm

**Conditional Flow Matching (CFM) policy for [robomimic](https://github.com/ARISE-Initiative/robomimic)** —
a standalone package that adds a rectified-flow / OT-CFM policy as a drop-in
alternative to Diffusion Policy, depending on robomimic rather than forking it.

It reuses robomimic's observation encoder and `ConditionalUnet1D` backbone, but
instead of learning to denoise it learns a **velocity field** `v(x_t, t, obs)`
along the straight-line path from Gaussian noise to action sequences:

```
x_t = (1 - (1 - σ_min)·t)·x₀ + t·actions      # linear interpolant, x₀ ~ N(0, I)
u_t = actions - (1 - σ_min)·x₀                # target velocity
loss = ‖ v_θ(x_t, t, obs) - u_t ‖²
```

At test time actions are generated by integrating the learned ODE with just
**10 Euler steps** (vs. 100 DDPM denoising steps), receding-horizon style exactly
like Diffusion Policy.

## Install

```bash
uv sync                 # resolves from uv.lock (robomimic pinned via the sibling fork)
# or, into an existing environment:
uv pip install -e .
```

Importing the package registers the algorithm with robomimic:

```python
import robomimic_cfm            # registers "flow_matching" algo + config
from robomimic.config import config_factory
config = config_factory(algo_name="flow_matching")
```

## Results (low_dim, proficient-human datasets)

**Success rate vs. Diffusion Policy**, matched budget (1000 epochs × 100
steps/epoch, batch 256), same encoder + `ConditionalUnet1D` backbone.
**Mean ± std over 3 seeds**, best-checkpoint on rollout success (n = 20 rollouts
per eval → 5% resolution). CFM uses 10 Euler steps; DP uses DDPM-100.

| Task | Flow Matching | Diffusion Policy | BC-RNN ref |
|---|---|---|---|
| Lift | **100.0 ± 0.0** | 100.0 ± 0.0 | ~100 |
| Can | **100.0 ± 0.0** | 100.0 ± 0.0 | ~100 |
| Square | **56.7 ± 8.5** | 45.0 ± 7.1 | ~84 |
| Tool Hang | 75.0 ± 7.1 | **80.0 ± 7.1** | ~67 |
| Transport | 81.7 ± 2.4 | **86.7 ± 8.5** | ~71 |

CFM is competitive with Diffusion Policy across the board — matched on the easy
tasks, clearly ahead on **Square** (the hardest precision task, +11.7 pts), and
slightly behind on Tool Hang / Transport.

**Inference speed** — per action-chunk latency on the identical 65M-param UNet
(RTX PRO 6000, 100 trials):

| Sampler | Latency (ms/chunk) | Speedup vs DDPM-100 |
|---|---|---|
| FM Euler, 1 step | **3.11 ± 0.02** | 94.4× |
| FM Euler, 5 steps | 13.44 ± 0.09 | 21.9× |
| FM Euler, 10 steps *(default)* | 26.68 ± 0.14 | **11.0×** |
| FM midpoint, 5 steps | 26.92 ± 0.14 | 10.9× |
| DP DDIM, 10 steps | 28.91 ± 0.10 | 10.2× |
| DP DDPM, 100 steps | 293.72 ± 0.61 | 1.0× |

CFM is **11× faster** than the standard DDPM-100 sampler at its default quality
setting. Against DDIM-10 (same number of network evaluations) it is ≈wall-clock
parity — its edge there is quality retained at few steps, not raw speed.

## Layout

```
robomimic_cfm/
  __init__.py            # import-time registration of the algo + config
  flow_matching.py       # FlowMatchingUNet (PolicyAlgo), registered as "flow_matching"
  config.py              # FlowMatchingConfig (BaseConfig, ALGO_NAME="flow_matching")
  exps/templates/flow_matching.json   # generated config template for train.py
tests/test_flow_matching.py           # 5 variants through train.py + reload + rollout
scripts/                              # train / eval / render / inference-benchmark
configs/                              # per-task + 3-seed benchmark configs (reproduction)
docs/CONDITIONAL_FLOW_MATCHING.md     # design notes
```

## Design

See [docs/CONDITIONAL_FLOW_MATCHING.md](docs/CONDITIONAL_FLOW_MATCHING.md).

Note: actions must be normalized to `[-1, 1]` (standard robomimic datasets
already are). The cosine LR schedule uses `step_every_batch=True` — if you write
your own training loop, step the scheduler **per gradient step** or the LR stays
at 0 through warmup.
